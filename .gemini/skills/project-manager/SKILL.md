# Project Manager Skill

This skill provides expert guidance for managing software projects, from initial planning to final delivery.

## Core Competencies

- **Implementation Planning**: Breaking down complex features into actionable tasks.
- **Scope Management**: Defining what is (and isn't) part of a project to avoid scope creep.
- **Dependency Management**: Identifying and resolving technical and resource blockers.
- **Agile Methodologies**: Scrum, Kanban, and Sprint planning.
- **Risk Assessment**: Identifying potential issues early and creating mitigation strategies.
- **Reporting**: Providing clear, concise progress updates to stakeholders.

## Common Workflows

### Creating an Implementation Plan

1. Define the high-level objective and success criteria.
2. Break down the project into logical milestones.
3. For each milestone, list specific tasks with clear "Definition of Done" (DoD).
4. Identify dependencies between tasks.
5. Estimate complexity (Story Points or T-Shirt sizing).

### Task Breakdown (Decomposition)

- **Requirement Analysis**: Translate user needs into technical requirements.
- **Task Granularity**: Ensure tasks are small enough to be completed within 1-2 days.
- **Acceptance Criteria**: Define specific tests or conditions each task must meet.

## Best Practices

- **SMART Goals**: Specific, Measurable, Achievable, Relevant, Time-bound.
- **Documentation First**: Ensure requirements and designs are documented before coding starts.
- **Iterative Delivery**: Deliver value in small, frequent increments.
- **Communication**: Maintain high visibility of progress and blockers.
- **Pragmatism**: Balance speed, quality, and scope according to project needs.
